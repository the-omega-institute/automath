\section{Reproducibility details and executable scripts}\label{app:pseudocode}

All reproducibility assets for Figure~\ref{fig:validation} live in the repository's \texttt{scripts/} directory.
The single entry point is \texttt{scripts/generate\_validation\_figure.py}, which depends only on Python~3.11, NumPy~1.26, Matplotlib~3.8, SymPy~1.12, and mpmath~1.3.
Every command below was run under that configuration.

\subsection{Bundled script for Figure~\texorpdfstring{\ref{fig:validation}}{1}}

The figure in Section~\ref{sec:verification} is generated by the Python script
\[
\texttt{scripts/generate\_validation\_figure.py}.
\]
From the paper directory, the figure can be rebuilt by running

\begin{verbatim}
python3 scripts/generate_validation_figure.py
\end{verbatim}

The script writes \texttt{figures/zero\_locus\_validation.pdf}, constructs \(Z_m(y)\) from the recurrence \eqref{eq:recurrence}, and computes spectral roots from \eqref{eq:spectral-poly} using \texttt{numpy}, \texttt{matplotlib}, and \texttt{mpmath}.
For the center panel it exploits the exact factorization $(y+1)^2\mid Z_m(y)$ proved in Proposition~\ref{prop:double-zero}: the quotient $Z_m(y)/(y+1)^2$ is computed symbolically (using integer arithmetic on the recurrence coefficients), and then its zeros are found numerically, so the double zero at $y=-1$ is an exact algebraic consequence rather than a numerical insertion.

\subsection{Single-point evaluation of \texorpdfstring{\(Z_m(y)\)}{Z\_m(y)}}

\begin{verbatim}
Input: m in N, y in C.

Initialize:
  Z0 = 1
  Z1 = y + 1
  Z2 = (y + 1)^2
  Z3 = (y + 1)^3

For k = 4,5,...,m:
  Zk = Z{k-1} + (2y+1) Z{k-2} - Z{k-3} - y(y+1) Z{k-4}

Output: Zm.
\end{verbatim}

\subsection{Rigorous interval isolation of the subdominant real branch point \texorpdfstring{\(\ySub\)}{ySub}}

Let \(c(y)=256y^3+411y^2+165y+32\).
Since \(c\) has exactly one real root, any rational interval \((\alpha,\beta)\) with
\(c(\alpha)c(\beta)<0\) certifies \(\ySub\in(\alpha,\beta)\).

\begin{verbatim}
Input: rationals alpha < beta.
Compute c(alpha), c(beta).
If c(alpha)c(beta) < 0:
  (alpha, beta) is a rigorous enclosure of y_sub.
  Repeat bisection to refine to desired precision.
\end{verbatim}

\subsection{Higher-precision checks near \texorpdfstring{\(y=0,1,\ySub\)}{y=0,1,ySub}}\label{subsec:higher-precision}

The bundled figure script uses IEEE double precision.
To cross-check the modulus ordering near the three delicate real points, one may run the following \(80\)-digit computation with \texttt{mpmath}; it can be pasted directly into a Python~3.11 shell:

\begin{verbatim}
from mpmath import mp

mp.dps = 80

def pi_coeffs(y):
    return [mp.mpf('1'), mp.mpf('-1'), -(2*y + 1), mp.mpf('1'), y*(y + 1)]

def ordered_moduli(y):
    roots = mp.polyroots(pi_coeffs(y), maxsteps=200, extraprec=120)
    return sorted((abs(root) for root in roots), reverse=True)

y_sub = mp.findroot(lambda yy: 256*yy**3 + 411*yy**2 + 165*yy + 32,
                    (mp.mpf('-1.2'), mp.mpf('-1.0')))

test_points = {
    "near_0": mp.mpf('1.0e-40'),
    "near_1_left": mp.mpf('1') - mp.mpf('1.0e-30'),
    "near_1_right": mp.mpf('1') + mp.mpf('1.0e-30'),
    "near_y_sub_left": y_sub - mp.mpf('1.0e-25'),
    "near_y_sub_right": y_sub + mp.mpf('1.0e-25'),
}

for name, y in test_points.items():
    moduli = ordered_moduli(y)
    print(name, moduli[0] - moduli[1])
\end{verbatim}

This computation gives a high-precision consistency check that the dominant-modulus gap is positive on both sides of \(y=1\) and remains positive near \(y=\ySub\), while the gap closes only at the endpoint \(y=0\).

\subsection{Dominant-modulus gap and zero-locus sampling}

For a fixed sample point \(y\in\CC\), compute the four roots of \(\Pi(\lambda,y)=0\),
sort them by decreasing modulus,
and record the dominant-modulus gap
\[
g(y)=|\lambda_{(1)}(y)|-|\lambda_{(2)}(y)|.
\]
Small values of \(g(y)\) approximate the dominant equimodular set predicted by the Beraha--Kahane--Weiss mechanism.
Separately, for a fixed \(m\), one may compute the roots of the polynomial \(Z_m(y)\) and compare them with the sampled small-gap region.

\begin{verbatim}
Input: grid points y in C, system size m.

For each sample point y:
  Compute the four roots lambda_i of Pi(lambda,y)=0.
  Sort them so |lambda_(1)| >= |lambda_(2)| >= |lambda_(3)| >= |lambda_(4)|.
  Record g(y) = |lambda_(1)| - |lambda_(2)|.

Compute the polynomial Z_m(y) from the recurrence and find its zeros.
Plot:
  - points where g(y) is small;
  - zeros of Z_m(y);
  - exact markers y = 0, y = 1, y = y_sub, y = -1.
\end{verbatim}

Figure~\ref{fig:validation} is produced in IEEE double precision (\texttt{numpy.float64}/\texttt{complex128}).
Both the quartic spectral polynomial and the polynomials \(Z_m(y)\) are solved with \texttt{numpy.roots}.
The double precision computation suffices for the global plot, but near the three delicate branch points the \(80\)-digit \texttt{mpmath} check in Section~\ref{subsec:higher-precision} provides a high-precision consistency check of the modulus ordering.

The real-axis modulus plot samples \(1500\) equally spaced points on \([-1.6,1.2]\), and the spectral roots are ordered by decreasing modulus
\[
\abs{\lambda}_{(1)}\ge \abs{\lambda}_{(2)}\ge \abs{\lambda}_{(3)}\ge \abs{\lambda}_{(4)}.
\]

The unique real root \(y_{\mathrm{sub}}\) of \(256y^3+411y^2+165y+32\) is selected by the criterion \(\abs{\Im y}<10^{-10}\).
Negative real zeros in the endpoint-scaling plot are selected by \(\abs{\Im y}<10^{-8}\) together with \(-1<\Re y<0\), and the right panel uses \(m=80,120,200\) with the first four zeros nearest \(0\).
