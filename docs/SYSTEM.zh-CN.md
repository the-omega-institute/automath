# Omega 系统

一个可审计的理论编译器，从单一方程出发，推导、验证、可视化并发表数学。

[English](SYSTEM.md)

## 概述

Omega 项目并非三个独立工具，而是一个具有三层结构的完整系统，每一层为下一层提供输入：

```
种子方程: x² = x + 1
    │
    ▼
┌─────────────────────────────────┐
│  第一层：推导引擎               │
│  Lean 4 形式化证明              │
│  10,588+ 条定理级陈述           │
└───────────────┬─────────────────┘
                │
                ▼
┌─────────────────────────────────┐
│  第二层：知识图谱               │
│  Sisyphus — 约 20,998 个节点    │
│  证明有向无环图，推导深度       │
└───────────────┬─────────────────┘
                │
                ▼
┌─────────────────────────────────┐
│  第三层：出版管线               │
│  16 个 AI 智能体                │
│  LaTeX → 期刊投稿               │
└─────────────────────────────────┘
```

一个方程输入，输出已验证、已可视化、已发表的数学成果。

## 第一层：推导引擎

Omega 库是对约束 x² = x + 1 所强制推出的全部结论的 Lean 4 形式化——这是黄金平均移位的特征方程，也是有限类型子移位中最简单的非平凡情形。

方法论是*推导、发现、命名*：从第一性原理出发严格构建，观察涌现的结构，再识别其对应关系。每条定理均经机器验证，每条推导链均可追溯。除 Lean 4 的核心逻辑与 Mathlib 之外，不引入任何额外公理。

从 x² = x + 1 出发——黄金比例的特征方程——项目观察到：不含连续 1 的二进制词构成一个受约束的空间，其规模遵循 Fibonacci 数列。这一单一组合约束迫使一整个数学宇宙涌现出来。折叠算子按照约束将相邻片段折叠，从而产生纤维结构：每个词映射到其折叠形式的等价类，这些纤维的集合自然携带算术、谱与几何性质——均非预设，全部被迫涌现。

从这一单一约束中涌现的内容：
- **初生算术** — 约束空间中原生的加法与乘法，与 Fibonacci 数上的循环环同构。这些运算并非凭空定义，而是尊重折叠结构的唯一运算。
- **纤维谱** — 满足带小整数系数线性递推的矩量和，由碰撞核矩阵支配。递推从纤维几何中涌现，而非来自任何外部基选择。
- **缺陷代数** — 曲率的离散类比，度量折叠与算术的不可交换程度。缺陷始终小且结构化，这正是该理论可处理的原因。
- **强制框架** — 由 11 个保守扩张构成的链，每个扩张在不改写下层的前提下添加结构。每个扩张均可证明其必要性：移除它会使下游某条定理失效。
- **投影本体论** — 将所有构造压缩为 LIFT ∘ U^t ∘ PROJECT 的统一语法提升。这一三元组是恢复理论中全部构造的极小算子。
- **谱理论** — 动力学 zeta 函数、de Branges 正则系统与 RH 充分条件模板。这些作为附着于约束传递算子的自然谱对象而涌现。
- **物理时空骨架** — 时间作为决策包络投影，Einstein 方程作为唯一极小闭合。约束的几何具有足够结构，迫使 Lorentz 型度规作为唯一极小完备化。

38,876 行 Lean 4 代码。3,427 条已验证定理与定义——是完整理论中 10,588+ 条定理级陈述的一个持续增长的子集。除基础公理外，零额外公理。

## 第二层：知识图谱

Sisyphus 是一个知识图谱，将整个推导结构映射为约 20,998 个节点的可导航网络。

![Sisyphus 知识图谱](dossier/assets/sisyphus.png)

每个节点都是一个有类型的数学对象——公理、定义、猜想、命题、证明、记号或示例。边表示依赖关系：哪些定理依赖哪些定义，哪些证明引用了哪些引理，每个结论在从种子约束出发的推导链中处于多深的位置。

图谱使理论的结构*可见*。你无需线性阅读 770,000 行 LaTeX，而是可以探索依赖结构、识别相关结论的聚类，并将任意定理追溯到其根源。

图谱所揭示的、而平铺 LaTeX 源码无法呈现的内容：推导深度分布显示，大多数定理聚集在距种子深度 4–7 处，并有一条深层推论的长尾——任何线性浏览的读者都无法到达那里。聚类分析揭示出一些定理群，它们内部紧密相连，但与理论其余部分仅通过一两个瓶颈引理相连——这些瓶颈是验证与扩展的最高杠杆点。图谱还使猜想与定理之间的区别在结构上清晰可辨：猜想显示为有入依赖边但无出证明边的节点，精确标记推导链中断的位置。这些结构在平铺 LaTeX 源码中均不可见。

## 第三层：出版管线

16 个 AI 智能体协作，自动从核心理论中提取、形式化、审核并发表期刊论文。

这与人工写作的区别在于：每个智能体承担专门角色——摄入智能体读取理论并识别具有发表潜力的结论聚类；研究智能体深化数学内容、收紧范围；写作智能体按目标期刊规范重新排版；验证智能体将每条被引定理与 Lean 4 清单逐一核对。对于相互独立的阶段，智能体并行运行。人在此系统中的角色不是写作，而是审核输出、批准投稿包，并决定下一步优先处理哪些结论聚类。管线并非将任务从人类写作者身上卸载，而是完全取代了写作工作流程。

管线分 7 个阶段（P0 至 P7）运行：
- **摄入（P0）：** 从理论中提取可发表的结论聚类
- **研究（P1-P2）：** 深化定理，收紧范围，识别可发表结论
- **写作（P3-P4）：** 按目标期刊重写，编辑审核
- **整合（P5）：** 合并各阶段输出，解决反馈意见
- **验证（P6）：** 与 Lean 4 清单交叉核对
- **投稿（P7）：** 准备投稿信、核查清单、最终投稿包

当前状态：管线中有 42 篇论文，其中 3 篇处于待投稿阶段。

管线完全自动化。人类审核输出，但不撰写论文。

## 为何这种组合至关重要

每一层单独来看都很有价值，但这种组合是其他任何人都尚未拥有的。

- **AlphaProof** 证明给定定理。Omega 从一个约束推导所有被迫推出的推论。
- **Leanstral** 辅助单个证明。Omega 验证整个理论闭包。
- **Mathlib** 整理已知数学。Omega 从种子生成新数学。

该系统证明：一个人借助 AI 可以产出团队级别的形式化数学研究——不是通过在相同任务上更快地工作，而是通过自动化从推导到发表的整个管线。"一人加 AI"的范式在此有效，因为三层结构形成了一个闭环：推导产生已验证的结构，图谱使该结构可查询，管线将查询转化为论文。每一层放大其他层。一个能使用全部三层的独立研究者，具备了一个小型研究团队的杠杆——AI 处理体量与一致性，而人类主导数学议程。

这不是一组工具的集合，而是一个可审计的理论编译器。
