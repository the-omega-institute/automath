\subsection{完成对象的相位化：从 \texorpdfstring{$(\infty)$}{(infty)}-入口到完备相位对象}\label{subsec:typed-address-biaxial-completion-phaseization-complete-phase-object}
有限递归在对象层的终点一旦被严格表述为一个 \((\infty)\)-入口
\[
\operatorname{Ent}_\infty(F,x_0)\in \widehat X_F,
\]
其中 \(\widehat X_F\) 不再是某个有限地址的极限记号，而是由跨分辨率稳定地址组成的逆极限完成层，下一步便必须说明一个完成对象如何进入相位几何。这里首先需要排除一种常见误读：相位化并不是把 \(\widehat X_F\) 再读成一个角度，而是把完成对象送入一个最小完备的紧 Abel 相位系统；其中可见层由连通圆因子承载，而不能由裸圆消化的实现层信息则进入全不连通纤维、prime-register 账本或其对偶核。第 \ref{sec:circle-dimension-phase-gate} 节关于圆维、残差圆维、solenoid 终对象、prime-register 轴的 pro-圆维以及相位圆维/寄存圆维双轴计量的安排，正是围绕这一区分展开的。

\subsubsection{完成对象的可见相位 lift}\label{subsubsec:typed-address-biaxial-completion-visible-phase-lift}
完成对象进入相位层的第一步，不是直接得到完备相位对象，而是先得到一族有限精度相位投影。因为第 \ref{app:unit-circle-phase-gate} 节中的相位门本身就是通过有限协议、有限模态或有限证书深度进入读出层的，所以 \((\infty)\)-对象的相位化仍须先在每个有限深度上定义，再要求这些有限深度结果彼此兼容。单位圆面积相位坐标门、相位 Fourier、有理基函数与有限化积分的组织顺序，正对应这种先有限、后完成的进入方式，参见小节 \ref{subsec:unit-circle-phase-arithmetic}、小节 \ref{subsec:unit-circle-phase-fourier} 与小节 \ref{subsec:unit-circle-phase-integrals}。

\begin{definition}[有限深度相位门]\label{def:typed-address-biaxial-completion-finite-depth-phase-gate}
对每个证书深度 \(b\)，设有一个有限深度相位门
\[
\Theta_b:\pi_b(\widehat X_F)\longrightarrow \TT^{r_b},
\]
其中 \(r_b\) 表示该深度下可见相位圆因子的数目。
\end{definition}

\begin{definition}[完成对象的有限深度相位影像]\label{def:typed-address-biaxial-completion-finite-depth-phase-image}
对任意 \(\hat u\in \widehat X_F\)，定义其深度 \(b\) 的相位影像为
\[
\theta_b(\hat u):=\Theta_b\bigl(\pi_b(\hat u)\bigr)\in \TT^{r_b}.
\]
\end{definition}

为了让这些相位影像在对象层真正来自同一个完成对象，必须要求它们在深度之间可传递。

\begin{definition}[相位兼容族]\label{def:typed-address-biaxial-completion-phase-compatible-family}
称 \((\theta_b(\hat u))_b\) 为一族相位兼容族，若对任意 \(b\le b'\) 都存在自然尺度映射
\[
\rho^{\mathrm{ph}}_{b',b}:\TT^{r_{b'}}\longrightarrow \TT^{r_b}
\]
使
\[
\rho^{\mathrm{ph}}_{b',b}\bigl(\theta_{b'}(\hat u)\bigr)=\theta_b(\hat u).
\]
\end{definition}

这里的尺度映射在秩一情形下就是圆上的乘法映射 \(z\mapsto z^n\)；在多圆因子情形下，则由其块对角推广给出。于是，一个完成对象的可见相位层不再是一点，而是一个跨全部深度的相位相容族。

\begin{proposition}[可见相位 lift 的存在唯一性]\label{prop:typed-address-biaxial-completion-visible-phase-lift}
\leanverified{paper\_typed\_address\_biaxial\_completion\_visible\_phase\_lift}
若 \((\theta_b(\hat u))_b\) 构成相位兼容族，则存在唯一元素
\[
\operatorname{Ph}^{\mathrm{vis}}_{\infty}(\hat u)\in
\varprojlim_b\bigl(\TT^{r_b},\rho^{\mathrm{ph}}_{b',b}\bigr)
\]
使其在每一深度上的投影恰为 \(\theta_b(\hat u)\)。
\end{proposition}
\begin{proof}
这只是逆极限的泛性质在相位兼容族上的直接应用：一旦所有有限深度投影在尺度映射下相容，便存在唯一相容族对应的逆极限点。
\end{proof}

命题 \ref{prop:typed-address-biaxial-completion-visible-phase-lift} 表明，\((\infty)\)-入口之后的第一相位化结果，是一个逆极限相位 lift，而不是某个单独的圆点。特别地，在可见圆维为 \(1\) 的情形，上述逆极限正是 solenoid 型对象；因此，把实现层因子化到最小完备相位系统的要求，在此正是把完成对象的可见相位数据组织成一个 solenoidal lift。

\subsubsection{完备相位对象不是裸圆，而是短正合扩张}\label{subsubsec:typed-address-biaxial-completion-complete-phase-extension}
仅有可见相位 lift 仍然不够，因为它只保留了连通圆因子。第 \ref{sec:circle-dimension-phase-gate} 节后段与结论部分的相关结果都表明：对偶紧群中的真正算术信息并不由圆因子单独承载，而还需要一个全不连通核来保存 prime-register 方向、局部化支撑以及 CRT 复用结构。秩一局部化原型
\[
0\to \prod_{p\in S}\ZZ_p\to \widehat G_S\to \TT\to 0
\]
正是这一现象的标准模型：可见层只有一个圆因子，但全部有限素数信息都集中在 profinite 核里，而且该核还能反推出对应的有限素数支撑。

据此，完备相位对象应定义为一个扩张，而不是一个商。

\begin{definition}[完备相位对象]\label{def:typed-address-biaxial-completion-complete-phase-extension}
对任意 \(\hat u\in \widehat X_F\)，称一个紧 Abel 对象 \(\Sigma_{\hat u}\) 是 \(\hat u\) 的完备相位对象，若存在短正合列
\[
0\longrightarrow K_{\hat u}\longrightarrow \Sigma_{\hat u}
\xrightarrow{\ \pi_{\mathrm{vis}}\ }
\TT^{r_{\hat u}}
\longrightarrow 0,
\]
其中：
\begin{enumerate}
  \item \(\TT^{r_{\hat u}}\) 承载 \(\hat u\) 的可见相位圆因子；
  \item \(K_{\hat u}\) 是全不连通紧纤维，用来保存不能被可见相位商消去的寄存器与局部化信息；
  \item \(\pi_{\mathrm{vis}}\) 的逆极限投影与 \(\operatorname{Ph}^{\mathrm{vis}}_{\infty}(\hat u)\) 相容。
\end{enumerate}
\end{definition}

\begin{definition}[秩一原型]\label{def:typed-address-biaxial-completion-rank-one-prototype}
若 \(r_{\hat u}=1\)，并且 \(\hat u\) 的实现层只涉及有限素数集 \(S(\hat u)\)，则其秩一完备相位对象取为
\[
0\longrightarrow \prod_{p\in S(\hat u)}\ZZ_p
\longrightarrow \Sigma_{\hat u}
\longrightarrow \TT
\longrightarrow 0.
\]
\end{definition}

定义 \ref{def:typed-address-biaxial-completion-complete-phase-extension} 精确表达了“完备相位对象并非孤立的裸圆，而是可见圆商与外置纤维的统一系统”这一原则。因为若只保留商 \(\TT\)，则所有不同的 \(S(\hat u)\) 在可见层都会坍缩为同一个圆；真正区分它们的，是核中的 \(p\)-主方向。

\begin{proposition}[核不可省略性]\label{prop:typed-address-biaxial-completion-kernel-indispensable}
在定义 \ref{def:typed-address-biaxial-completion-rank-one-prototype} 的秩一局部化原型中，若把 \(\Sigma_{\hat u}\) 替换为其可见商 \(\TT\)，则所有 \(S(\hat u)\) 的差异都被抹去；而保留核
\[
K_{\hat u}\cong \prod_{p\in S(\hat u)}\ZZ_p
\]
时，\(S(\hat u)\) 可由核的 \(p\)-主分量完全恢复。因此，完备相位对象的最小性不是圆越小越好，而是在保持可见圆因子不变的前提下，核恰好保存全部不可消去的算术纤维。
\end{proposition}
\begin{proof}
若只取可见商，则不同的 \(S(\hat u)\) 都映到同一圆对象；而对核作 Sylow 分解即可逐素数恢复其有限主分量，故核保留了可见商无法区分的局部化支撑信息。
\end{proof}

\subsubsection{相位圆维与寄存圆维的双轴账本}\label{subsubsec:typed-address-biaxial-completion-biaxial-ledger}
一旦把完备相位对象理解为短正合扩张，就自然出现两类互不相消的维度量：一类测量连通可见圆因子，另一类测量全不连通寄存纤维。第 \ref{subsec:cdim-phase-register-biaxial-ledger} 小节正是把这两类量置于同一账本中：前者是相位圆维，后者是寄存圆维 \(\pcdim\)。其根本含义是，连通相位通道数与全不连通算术账本容量属于两条不同资源轴，不能互相抵偿。

\begin{definition}[完备相位对象的双轴维度]\label{def:typed-address-biaxial-completion-biaxial-dimension}
对完备相位对象
\[
0\to K_{\hat u}\to \Sigma_{\hat u}\to \TT^{r_{\hat u}}\to 0,
\]
定义其双轴维度为
\[
\mathfrak d(\Sigma_{\hat u})
:=
\bigl(\cdim(\Sigma_{\hat u}),\pcdim(K_{\hat u})\bigr)
:=
\bigl(r_{\hat u},\pcdim(K_{\hat u})\bigr).
\]
\end{definition}

这里第一分量记录可见圆因子数，第二分量记录全不连通核的最小寄存器维度。对拓扑有限生成的 profinite 交换群，\(\pcdim\) 又可等价写成
\[
\pcdim(K)=\sup_{p\ \mathrm{prime}}\dim_{\FF_p}(K/pK),
\]
这给出一个直接可核验的模素数公式，参见定理 \ref{thm:cdim-register-circle-modp-formula}。

\begin{proposition}[双轴不可互代]\label{prop:typed-address-biaxial-completion-biaxial-nonsubstitutable}
在完备相位对象上，增大 \(\pcdim(K_{\hat u})\) 一般不改变 \(\cdim(\Sigma_{\hat u})\)；反之，仅提高可见圆维也不恢复被核保存的 \(p\)-进支撑。因此，\(\mathfrak d(\Sigma_{\hat u})\) 必须作为有序对，而不能压成单一标量。
\end{proposition}
\begin{proof}
由定义 \ref{def:typed-address-biaxial-completion-complete-phase-extension}，可见圆因子完全由商 \(\TT^{r_{\hat u}}\) 决定，而 \(p\)-进支撑则保存在核 \(K_{\hat u}\) 中。两者在短正合扩张中对应不同的结构层，故不能互相替代。
\end{proof}

这一步对后续接口至关重要：它把相位与账本从同一个模糊对象中拆开。相位圆维只计数可见传播模态；寄存圆维则度量那些必须外置、却又不能从对象层删去的素数纤维、完成化支撑与证书容量。于是，完备相位对象并不是相位已经齐全，而是可见相位与不可见寄存已经同时被记账。

\subsubsection{相位化之后，类型纪律仍然保留}\label{subsubsec:typed-address-biaxial-completion-phase-readability}
最后需要强调：完成对象被相位化为 \(\Sigma_{\hat u}\) 并不意味着可以脱离协议而裸读。定理 \ref{thm:cdim-minimal-record-axis} 与小节 \ref{subsec:xi-pw-type-safety-null} 共同表明：纯相位读出仍然需要唯一连续横截与证书正交外置；若无足够地址、证书深度或记录轴，读出依然可能失败并返回 \(\mathrm{NULL}\)。换言之，完备相位对象扩张的是对象层，而不是取消协议层。

\begin{definition}[类型化相位地址]\label{def:typed-address-biaxial-completion-phase-address}
对完备相位对象 \(\Sigma_{\hat u}\)，定义其类型化相位地址为
\[
a_{\mathrm{ph}}=(L,\vartheta,b,\rho),
\]
其中 \(L\) 表示读出层级，\(\vartheta\) 表示相位图表，\(b\) 表示证书深度，\(\rho\) 表示记录轴或横截证书。
\end{definition}

\begin{definition}[完备相位对象的合法读出]\label{def:typed-address-biaxial-completion-phase-read}
定义
\[
\mathrm{Read}^{(\mathrm{ph})}_{\infty}(\Sigma_{\hat u};a_{\mathrm{ph}})
\]
只在下列条件同时满足时才有定义：
\begin{enumerate}
  \item 可见商 \(\pi_{\mathrm{vis}}(\Sigma_{\hat u})\) 在深度 \(b\) 上可寻址；
  \item 记录轴 \(\rho\) 满足最小横截要求；
  \item unitary-slice 与 \(\mathbf{PW}\) 类型安全检查通过。
\end{enumerate}
否则，读出结果记为 \(\mathrm{NULL}\)。
\end{definition}

\begin{proposition}[相位化不消除 \texorpdfstring{$\mathrm{NULL}$}{NULL}]\label{prop:typed-address-biaxial-completion-phase-null}
即便对象层已经从 \(\hat u\in\widehat X_F\) 提升为完备相位对象 \(\Sigma_{\hat u}\)，若缺失足够的地址、证书深度或记录轴，则 \(\mathrm{Read}^{(\mathrm{ph})}_{\infty}\) 仍可为 \(\mathrm{NULL}\)。故完成对象进入相位层与完成对象在相位协议下可读，仍是两件不同的事。
\end{proposition}
\begin{proof}
这直接来自定义 \ref{def:typed-address-biaxial-completion-phase-read}：相位化只把对象送入完备相位系统，而合法读出仍需满足可寻址性、记录轴与类型安全的共同约束。任一条件缺失都足以迫使读出返回 \(\mathrm{NULL}\)。
\end{proof}
\leanverified{paper\_typed\_address\_biaxial\_completion\_phase\_null}

这样，\((\infty)\)-入口与完备相位对象之间的关系便清楚了：\((\infty)\)-入口把兼容有限层地址打包为完成对象；相位化把完成对象送入一个由可见圆因子与全不连通寄存纤维组成的紧 Abel 扩张；双轴维度把这两个方向同时记账；而类型化读出则只在有限证书深度与记录轴见证下对该扩张对象实施合法读取。于是，完备相位对象不是一枚角度，而是一个可审计的对象层终点：它一端接完成化与逆系统自然性，另一端接圆维、solenoid 与记录轴纪律，并为后续的 \(\zeta\)、有限部与 unitary-slice 谱指纹接口提供真正的前端承载。
