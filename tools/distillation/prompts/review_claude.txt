Review criteria:
- Domain correctness and conservative mathematical strength.
- Local context fit in the target section.
- Concrete insertion target: reject writebacks aimed at main.tex wrappers or
  files that merely input other files.
- Do not run shell commands, inspect files, or use external tools. Review only
  the supplied writebacks and section context.
- Main-paper language: accepted LaTeX content must be formal academic Chinese,
  while preserving macros, labels, object names, and mathematical notation.
- Chinese writeback rule overrides target-file language. Do not require
  translation to English even when the supplied target context is mostly or
  entirely English; language mismatch with old context is not a blocker.
- Killo-golden section hygiene: reject snippets that open \part, \chapter,
  \section, \subsection, \subsubsection, \paragraph, or \subparagraph. The
  writeback must integrate into an existing concrete target context unless a
  later human paper-structuring step explicitly opens a necessary section.
- Killo-golden style hygiene: reject visible patch/change-log phrasing, manual
  prefixes such as "new result", "补充 A", "结论 1", "闭环 1", "新增", "本次",
  "本轮", and any timestamp or edit-history wording inside the LaTeX content.
- Body-file budget: reject any proposal that would force the target file past
  the 600-line gate.
- Application planner note: accepted blocks are inserted before an existing
  \endinput automatically. Do not reject solely because a target file ends
  with \endinput unless the proposed content itself emits \endinput or would
  be invalid before it.
- Notation and dependency consistency.
- Reject hard-coded human-facing result ordinals such as "Result 18" or
  "\u7ed3\u679c 18" when they duplicate or may collide with nearby target
  numbering. Also reject numbered displayed entry codes such as WZ-GP-18 or
  \mathsf{{nr}}=19. Stable LaTeX labels are the canonical identifiers.
- No duplication of existing nearby text.
- Integration quality: the writeback must help future Omega routing, not merely add commentary.
- Explicit separation between construction, rigidity, obstruction, closure, and heuristic language.
- DEPTH: writebacks must contain real theorem/lemma/definition environments with proofs or proof sketches. Prose-only output is UNACCEPTABLE.

Scoring:
1-3 means reject (fundamentally flawed or shallow filler).
4-6 means revise (structure exists but content is too thin or incorrect).
7-8 means acceptable with minor risk only when no mandatory revision remains.
9-10 means strong integration with deep mathematical content.

depth_assessment: "shallow" if only remarks/commentary without formal claims, "adequate" if definitions+lemmas present, "deep" if theorems with proofs.

Use verdict="accept" only if the writebacks may be applied as-is. If any mandatory correction is needed, verdict must be "revise" or "reject", score must be at most 6, and required_changes must be nonempty. For accept, required_changes must be empty; put nonblocking residual risks in issues only.
